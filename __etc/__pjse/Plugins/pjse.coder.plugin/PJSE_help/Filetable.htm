The PJSE Filetable

 

The PJSE Filetable does not coincide with the SimPE filetable exactly.  PJSE does understand your latest EP or SP installation via the SimPE settings, but not the extra paths you define in SimPE.   This is so that it doesn't waste memory on resources that are not connected with Simantics editing.   Extra paths and/or individual files can be defined to PJSE via \<SimpePath>\Data\Plugins\pjse.coder.plugin\packages.txt (case sensitive).  You can create the folder and file if they do not already exist.

Use one fresh line for each path or file defined.  Typical lines in packages.txt would look like the following

C:\Documents and Settings\Inge Jones\My Documents\EA Games\The Sims 2\Downloads\ Loose packages in Downloads
C:\Documents and Settings\Inge Jones\My Documents\EA Games\The Sims 2\Downloads\+ All packages in Downloads including those in subfolders
C:\Documents and Settings\Inge Jones\My Documents\WorkInProgress\CurrentProject\+ All packages and folders under a particular project folder
C:\Documents and Settings\Inge Jones\My Documents\WorkInProgress\HandyResources.package A particular package file

Please note we use a + to mean recurse, so it does not get confused with the more ambiguous *

 

Why would you want to define these extra paths and files?

Most people will never feel the need to do this at all, but some of the benefits are:

1. If you have created new semiglobals or globals that your current project needs to use (rare, but I have needed to do this on a couple of complex projects), and PJSE can find the package containing them via its filetable, you won't be told "invalid BHAV" on the line you call those globals from - it will be able to display the name and show you how many parameters the called BHAV expects etc.

2. Maxis did not distribute most of their BCON labels with objects.package.   People are finding out what various BCONs mean and are creating their own TRCN resources (BCON label files).  These can be placed as packages in a folder named by packages.txt, and then PJSE will use them to automatically display the meaning of the BCON wherever it appears in the file you have open.  There are other types of label files such as one for local variables and parameters (TPRP) which this applies to also.

If one of the PJSE plugins is not showing some resources you expect it to, then usually clicking the RFT (stands for Refresh File Table) button on the top bar of your plugin window will put that right.  Please remember this is not the same as the overall SimPE filetable refresh option which is found elsewhere on the menus.
