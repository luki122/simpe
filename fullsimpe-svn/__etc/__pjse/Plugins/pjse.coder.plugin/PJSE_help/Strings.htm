The String Editor Plugin (STR#,CTSS,TTAs)

 

String files can contain a set of strings translated into many languages.  The default is "English" which is also treated as the local language for United States.   Britain has a different English set.   When making your own object, unless you are multilingual, it is recommended that you only use the default (English) string set, and delete all the others (using Default Lang Only), to keep the file size small.   A player in any country that does not have a set of strings for that language will automatically see the default strings instead.   This also applies to individual  missing strings.   So if you are cloning say a chair which you want to carry on acting as a chair as well as having some additional new options - you can keep all the existing strings in the foreign languages, but add your new ones in your own language to the default set.    If you want to change the text for an existing option, also do that in the default set, and then clear that string out of the other language sets (using Default String Only).

On many types of string file - and I have yet to establish for sure which ones these apply to - if a string is missing from the file altogether, the game will look for the corresponding string in the semi-global set, and if that does not exist, will fall back to the global set.

 

Headers

These values refer to the textlist file as a whole

Filename Any descriptive title you like
Format Not sure, probably to do with how many strings you can have.  Probably best not to edit this.
Commit Updates the package file in memory to include edits to current resource.  No change made to file on disk.  If you don't commit, you can abandon edits to current resource by simply opening another resource.

Text input boxes

These values refer to the currently selected string.  While editing in these boxes you can select the next or previous string using the keyboard shortcuts Alt+D and Alt+U if you don't wish to lose focus by clicking on the string list.   The button to the right of each box opens up a nicer edit window in case you want to lay out your text over several lines and want to be able to see it all at once while doing so.    The up and down arrows near the String input box scroll through the strings without having to get RSI mousing down to the string list.

 

String This is the active text, for example what will display in a dialog box or the Buy catalog.
Desc This is for you to make your own reference notes.  You do not need to fill anything in here unless you want to.

 

Language Navigator - Go to / Add

This enables you to turn to any other language set easily..

Return to default language set immediately
Move to next language set
Move back one language set
Pick language to move to
Reveal default language for comparison All 5 columns are actually present in the display but if you tick this, their widths are automatically adjusted so that you can see another (non-editable) column, listing the default strings, so that you can use it for reference when you are translating strings into another language.
Reveal string descriptions All 5 columns are actually present in the display but if you tick this, the column widths will be adjusted so that you will see a small part of the description where present for any string you have displayed.  Columns may be resized.

 

Buttons

Many of these buttons will be disabled when a language set other than default is open, as it is vital to consider the affect of actions on the default set, predominantly.  When that is correct, then is the time to consider whether you want a localised translation for any language.

Export Lang Exports currently selected language to a text file.  Uses labels <-String-> and <-Desc-> (one pair of each for every string in the language set whether used or empty) and <-Comment-> (optional for programmers' comments).   The format of the output is deliberately non-technical looking so as not to scare off anyone who might be doing language translations for you. You can use a <-Comment-> block to remind your translator not to delete or rearrange any of the label blocks!
Import Lang Imports text file in the above format to currently selected file.  Text block beneath one label in the import file will be considered input to that field until the next label is reached.   Reserve unused strings or descriptions in the middle of the set by using labels with no text block under them.

Any language file for import containing extended characters (e.g. accented characters or two-byte charcters like Chinese) should be saved to UTF-8 format which will allow PJSE to import it correctly.
Append File Opens dialog where you can choose from available string files.  Appends strings across all language sets, appropriately renumbered
Default Lang Only Acts on entire file by wiping out all strings from all language sets other than default (English).
Clear <language> Deletes all strings in the current language set.  Not enabled when in default language as it would be unhelpful to do so.
Add String Adds string to end of set across all languages.  If an existing string was selected at the time, that will be copied (for the default string only), otherwise the string will be blank.
Delete String Deletes current string across *all* language sets, and renumbers subsequent lines to close up the gap.
Make Default Overwrites corresponding string of the default language set with selected string from current language set.
Default String Only Wipes out selected string across all language sets other than default
Copy String to All Copies currently selected string in currently selected language set to all languages replacing whatever they already had in that string position.  In most situations you don't need to do this, you just use "make default" and "default string only" which keeps the file size smaller.  However there are some cases where the game will not fall back to default reliably, and it is appropriate to use this choice for those.
Replace String Replace currently selected string with one from another string resource, across all language sets. Brings up a resource chooser where you can choose the source file and from there, the individual string.
